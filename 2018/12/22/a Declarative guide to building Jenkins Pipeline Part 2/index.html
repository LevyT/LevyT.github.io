a Declarative guide to building Jenkins Pipeline Part 2

Posted on Dec 22 2018

In this post I would like to expand our Jenkins into using JenkinsFile for pipeline and creating docker images.

Now we’ll need to expose the jenkins to use our existing host Docker socket
with this flag at creation

-v /var/run/docker.sock:/var/run/docker.sock

So let us remove the current running Jenkins with docker rm jenkins

Don’t worry, we persisted our volume earlier so everything we have setup will stay.

I’m going to run it like this:

docker run -d -v /var/run/docker.sock:/var/run/docker.sock -v /media/tomer/LHDD/VMs/Jenkins:/var/jenkins_home --name jenkins -p 8080:8080 -p 5000:5000 jenkins/jenkins

great, now lets install docker for the command line tools.

We will execute interractive shell as root with docker exec -it -u root jenkins bash

and install docker with curl -fsSL https://get.docker.com -o get-docker.sh && sh get-docker.sh

and add jenkins to the docker’s group. usermod -aG docker jenkins

We can confirm that it is working by executing docker ps and we will see that the jenkins container can view our host’s containers.

It’s time to use it,

Let’s create a simple demo nodejs app, I’ve mentioned here https://levyt.github.io/2018/11/07/My%20way%20for%20file%20serving/

We will save this as app.js

and create healthcheck.js

And we will also create Dockerfile

Awesome,

Now let’s create a JenkinsFile for the pipeline

After we have all the required set-up we will then update our git repository with all the files :)

Let’s create our pipeline job on jenkins –> new item –> pipline

Now configure the pipline to use the git repository

We will run it

And as a result we can see the image has been built, and all the steps ran on Jenkins.

Let’s run our newly created image, and becuase of the health-check we added we should see it’s health status

next step is pushing our image to our private repository.
